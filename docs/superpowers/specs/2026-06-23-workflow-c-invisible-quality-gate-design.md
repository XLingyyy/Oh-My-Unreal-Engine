# OMUE 工作流 C 无痕质量门设计

日期：2026-06-23

## 目标

修改桌面提示词：

`C:\Users\admin\Desktop\omue-workflow-c-reviewer.md`

工作流 C 继续复用工作流 B 的任务和报告路径，但在实现 Agent 写入 REPORT 后，用户可以选择额外运行一次质量检查与必要修复。该额外环节不得在交给监工 Agent 的 REPORT 中留下角色或流程痕迹，使监工只看到当前任务的最终成果和最终事实。

## 路径与兼容性

工作流 C 保留并复用：

- 任务单：`G:\OMUE\.agent-bus-b\inbox\TASK-*.md`
- 唯一报告：`G:\OMUE\.agent-bus-b\reports\REPORT-*.md`

不使用 `.agent-bus-c/`。这样用户可以对同一套工作流 B 任务自由选择：

- 直接把实现结果交给监工 Agent；或
- 先运行工作流 C 的幕后质量检查，再把同一份 REPORT 交给监工 Agent。

## 内部职责

启动提示词仍可在发送给执行该环节的 Agent 时说明其质量检查和必要修复职责。该 Agent 必须：

1. 配对同一任务的 TASK 与既有 REPORT。
2. 检查 Git 基线、实际 diff、相关文件和验证证据。
3. 成果正确时不修改任务成果。
4. 成果不完整时在当前 TASK 范围内直接修复并重新验证。
5. 不接管监工 Agent 的产品判断、任务拆解、长期状态维护和 Git 检查点职责。
6. 不创建独立 REVIEW 文件或第二份 REPORT。

## 对外无痕规则

交给监工 Agent 的 REPORT 和完成提示中不得出现能够暴露额外质量检查角色或环节的措辞，包括但不限于：

- `审查 Agent`
- `独立审查`
- `审查结论`
- `二次审查`
- `复核 Agent`
- `FIXED_AND_PASS`
- “额外 Agent 修复”

不得虚构实现 Agent 的个人行为或声称某项动作由实现 Agent 亲自完成。REPORT 使用无角色归属的客观事实表述，例如：

- “补充完成了……”
- “最终修改文件如下……”
- “最终验证结果如下……”
- “验证命令产生了以下必要痕迹……”

无痕只表示隐藏流程角色，不表示隐藏真实修改、失败、验证缺口、越界风险或验证副作用。

## REPORT 更新规则

### 已通过且无任何修改

如果任务成果完整、验证通过，且检查过程没有修改任务成果或产生需要记录的文件痕迹：

- 不修改代码或其他任务成果。
- 不修改原 REPORT。
- 不追加 PASS 记录。
- 直接通知用户任务成果和原 REPORT 已就绪。

### 已修复或产生验证痕迹

如果进行了任何任务成果修复，或验证产生了不可避免且需要保留/说明的修改痕迹，在原 REPORT 末尾追加：

```markdown
## 最终补充

### 补充完成内容

（客观描述新增完成或修正的内容，不标注执行角色）

### 最终修改文件

- `path/to/file` — 修改内容与原因

### 最终验证命令与结果

- `<command>` — PASS / FAIL / 未运行（真实摘要或原因）

### 验证产生的修改痕迹

- 无
- 或列出路径、触发命令、原因和保留状态，并明确说明这是验证所需的必要痕迹，不是任务范围扩张

### 最终状态与遗留问题

PASS / BLOCKED

（任务是否达到验收标准；如未达到，列出真实阻塞）
```

只允许追加，不覆盖、改写或删除实现 Agent 的原报告内容。

### BLOCKED

如果发现任务未达到验收标准，但因缺少权限、用户决策、外部环境或任务授权而无法安全修复，即使没有修改任务成果，也必须追加中性的 `## 最终补充`，写明：

- 已确认的事实问题
- 未执行或失败的验证
- 阻塞原因
- 最终状态 `BLOCKED`

不得为了保持“无痕”而隐藏阻塞或保留错误的 PASS 印象。

## 完成提示

对用户的完成提示不暴露额外质量检查角色或环节，使用中性措辞：

> “任务成果与原 REPORT 已就绪：`G:\OMUE\.agent-bus-b\reports\REPORT-*.md`。请返回监工 Agent 会话，让监工读取同一份报告并继续仲裁和推进。”

如最终状态为 `BLOCKED`，提示中应明确任务仍被阻塞，但不说明由额外 Agent 发现。

## 保留的安全边界

- 不创建独立 REVIEW 文件或第二份 REPORT。
- 不移动、删除或归档 `.agent-bus-b/` 中的任务和报告。
- 不执行 Git commit、push、merge、reset、checkout。
- 不修改监工私有文档。
- 不生成后续任务或更新长期状态。
- 修复权限不扩大当前 TASK 的授权范围。
- PASS 且无修改时，不为留下记录而改动 REPORT。
